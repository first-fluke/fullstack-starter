# Antigravity マルチエージェントスキルの使用方法

[English](./USAGE.md) | [한국어](./USAGE.ko.md) | [简体中文](./USAGE.cn.md) | 日本語

## クイックスタート

1. **Antigravity IDE で開く**
   ```bash
   antigravity open /path/to/subagent-orchestrator
   ```

2. **スキルが自動検出されます。** Antigravity は `.agent/skills/` をスキャンし、すべての利用可能なスキルをインデックス化します。

3. **IDE でチャット。** 構築したいものを説明してください。

---

## 使用例

### 例 1：シンプルな単一ドメインタスク

**入力：**
```
"Tailwind CSS を使用して、メールとパスワードフィールドを持つログインフォームコンポーネントを作成"
```

**発生すること：**
- Antigravity はこれが `frontend-agent` に一致することを検出
- スキルが自動的に読み込まれる（プログレッシブディスクロージャー）
- TypeScript、Tailwind、フォームバリデーション付きの React コンポーネントを取得

### 例 2：複雑なマルチドメインプロジェクト

**入力：**
```
"ユーザー認証付きの TODO アプリを構築"
```

**発生すること：**

1. **ワークフローガイドがアクティブ化** — マルチドメインの複雑性を検出
2. **PM エージェントが計画** — 優先度付きのタスク分解を作成
3. **エージェントマネージャー UI でエージェントを生成**：
   - バックエンドエージェント：JWT 認証 API
   - フロントエンドエージェント：ログインと TODO UI
4. **エージェントが並列で動作** — 出力をナレッジベースに保存
5. **調整** — `.gemini/antigravity/brain/` で整合性をレビュー
6. **QA エージェントがレビュー** — セキュリティ/パフォーマンス監査
7. **修正と反復** — 修正を加えてエージェントを再生成

### 例 3：バグ修正

**入力：**
```
"バグがある — ログインをクリックすると 'Cannot read property map of undefined' と表示される"
```

**発生すること：**

1. **debug-agent がアクティブ化** — エラーを分析
2. **根本原因が判明** — コンポーネントがデータ読み込み前に `todos` をマッピング
3. **修正が提供される** — ローディング状態と null チェックを追加
4. **回帰テストが作成される** — バグが再発しないことを保証
5. **類似パターンが検出される** — 先制的に他の 3 つのコンポーネントを修正

### 例 4：CLI ベースの並列実行

```bash
# 単一エージェント
./scripts/spawn-subagent.sh backend "Implement JWT auth API" ./backend

# 並列エージェント
./scripts/spawn-subagent.sh backend "Implement auth API" ./backend &
./scripts/spawn-subagent.sh frontend "Create login form" ./frontend &
./scripts/spawn-subagent.sh mobile "Build auth screens" ./mobile &
wait
```

**リアルタイム監視：**
```bash
# ターミナル（別のターミナルウィンドウ）
bun run dashboard

# またはブラウザ
bun run dashboard:web
# → http://localhost:9847
```

## リアルタイムダッシュボード

### ターミナルダッシュボード

```bash
bun run dashboard
```

`fswatch` (macOS) または `inotifywait` (Linux) を使用して `.serena/memories/` を監視。セッションステータス、エージェント状態、ターン、最新アクティビティを含むライブテーブルを表示。メモリファイルが変更されると自動的に更新。

**要件：**
- macOS: `brew install fswatch`
- Linux: `apt install inotify-tools`

### Web ダッシュボード

```bash
bun install          # 初回のみ
bun run dashboard:web
```

ブラウザで `http://localhost:9847` を開きます。機能：

- **リアルタイム更新** WebSocket 経由（イベント駆動、ポーリングではない）
- **自動再接続** 接続が切断された場合
- **Serena テーマ UI** 紫色のアクセントカラー
- **セッションステータス** — ID と running/completed/failed 状態
- **エージェントテーブル** — 名前、ステータス（色付きドット付き）、ターンカウント、タスク説明
- **アクティビティログ** — 進捗ファイルと結果ファイルからの最新の変更

サーバーは chokidar とデバウンス (100ms) を使用して `.serena/memories/` を監視。変更されたファイルのみが読み取りをトリガー — 完全な再スキャンは行われません。

## 主要コンセプト

### プログレッシブディスクロージャー
Antigravity は自動的にリクエストとスキルをマッチング。スキルを手動で選択することはありません。必要なスキルのみがコンテキストに読み込まれます。

### トークン最適化されたスキル設計
各スキルは最大のトークン効率のために 2 層アーキテクチャを使用：
- **SKILL.md** (~40 行)：アイデンティティ、ルーティング、コアルール — 即座に読み込み
- **resources/**：実行プロトコル、例、チェックリスト、エラープレイブック — オンデマンドで読み込み

共有リソースは `_shared/`（スキルではない）にあり、すべてのエージェントが参照：
- 4 ステップワークフローのチェーンオブソート実行プロトコル
- ミッドティアモデルガイダンス用の Few-shot 入出力例
- "3 ストライク"エスカレーションのエラー復旧プレイブック
- 構造化されたマルチステップ分析の推論テンプレート
- Flash/Pro モデルティアのコンテキスト予算管理
- `verify.sh` による自動検証
- クロスセッションの教訓の蓄積

### エージェントマネージャー UI
Antigravity IDE のミッションコントロールダッシュボード。エージェントを生成、ワークスペースを割り当て、インボックスで監視、アーティファクトをレビュー。

### ナレッジベース
エージェント出力は `.gemini/antigravity/brain/` に保存。計画、コード、レポート、調整メモを含む。

### Serena メモリ
`.serena/memories/` の構造化されたランタイム状態。オーケストレーターはセッション情報、タスクボード、エージェントごとの進捗、結果を書き込みます。ダッシュボードはこれらのファイルを監視して監視を行います。

### ワークスペース
エージェントは競合を避けるために別々のディレクトリで動作可能：
```
./backend    → バックエンドエージェントワークスペース
./frontend   → フロントエンドエージェントワークスペース
./mobile     → モバイルエージェントワークスペース
```

## 利用可能なスキル

| スキル | 自動アクティブ化条件 | 出力 |
|-------|-------------------|--------|
| workflow-guide | 複雑なマルチドメインプロジェクト | ステップバイステップのエージェント調整 |
| pm-agent | "plan this"、"break down" | `.agent/plan.json` |
| frontend-agent | UI、コンポーネント、スタイリング | React コンポーネント、テスト |
| backend-agent | API、データベース、認証 | API エンドポイント、モデル、テスト |
| mobile-agent | モバイルアプリ、iOS/Android | Flutter 画面、状態管理 |
| qa-agent | "review security"、"audit" | 優先順位付き修正の QA レポート |
| debug-agent | バグレポート、エラーメッセージ | 修正済みコード、回帰テスト |
| orchestrator | CLI サブエージェント実行 | `.agent/results/` の結果 |

## ワークフローコマンド

Antigravity IDE チャットでこれらを入力してステップバイステップのワークフローをトリガー：

| コマンド | 説明 |
|---------|-------------|
| `/coordinate` | エージェントマネージャー UI によるマルチエージェントオーケストレーション |
| `/orchestrate` | 自動化された CLI ベースの並列エージェント実行 |
| `/plan` | API コントラクト付き PM タスク分解 |
| `/review` | フル QA パイプライン（セキュリティ、パフォーマンス、アクセシビリティ、コード品質） |
| `/debug` | 構造化されたバグ修正（再現 → 診断 → 修正 → 回帰テスト） |

これらは**スキル**（自動アクティブ化）とは別です。ワークフローにより、マルチステッププロセスを明示的に制御できます。

---

## 典型的なワークフロー

### ワークフロー A：単一スキル

```
あなた: "ボタンコンポーネントを作成"
  → Antigravity が frontend-agent を読み込む
  → すぐにコンポーネントを取得
```

### ワークフロー B：マルチエージェントプロジェクト（自動）

```
あなた: "認証付きの TODO アプリを構築"
  → workflow-guide が自動的にアクティブ化
  → PM エージェントが計画を作成
  → エージェントマネージャーでエージェントを生成
  → エージェントが並列で動作
  → QA エージェントがレビュー
  → 問題を修正し、反復
```

### ワークフロー B-2：マルチエージェントプロジェクト（明示的）

```
あなた: /coordinate
  → ステップバイステップのガイド付きワークフロー
  → PM 計画 → 計画レビュー → エージェント生成 → 監視 → QA レビュー
```

### ワークフロー C：バグ修正

```
あなた: "ログインボタンが TypeError をスロー"
  → debug-agent がアクティブ化
  → 根本原因分析
  → 修正 + 回帰テスト
  → 類似パターンをチェック
```

### ワークフロー D：ダッシュボード付き CLI オーケストレーション

```
ターミナル 1: bun run dashboard:web
ターミナル 2: ./scripts/spawn-subagent.sh backend "task" ./backend &
             ./scripts/spawn-subagent.sh frontend "task" ./frontend &
ブラウザ:     http://localhost:9847 → リアルタイムステータス
```

## ヒント

1. **具体的に** — "JWT 認証、React フロントエンド、FastAPI バックエンドで TODO アプリを構築" は "アプリを作る" より良い
2. **マルチドメインプロジェクトではエージェントマネージャーを使用** — 1 つのチャットですべてを行おうとしない
3. **ナレッジベースをレビュー** — `.gemini/antigravity/brain/` で API 整合性を確認
4. **再生成で反復** — 指示を改良し、最初からやり直さない
5. **ダッシュボードを使用** — `bun run dashboard` または `bun run dashboard:web` でオーケストレーターセッションを監視
6. **ワークスペースを分離** — 各エージェントに独自のディレクトリを割り当てる

## トラブルシューティング

| 問題 | 解決策 |
|---------|----------|
| スキルが読み込まれない | `antigravity open .`、`.agent/skills/` を確認、IDE を再起動 |
| エージェントマネージャーが見つからない | 表示 → エージェントマネージャーメニュー、Antigravity 2026+ が必要 |
| エージェント出力が互換性ない | ナレッジベースで両方をレビュー、修正を加えて再生成 |
| ダッシュボード："No agents" | メモリファイルがまだ作成されていない、まずオーケストレーターを実行 |
| Web ダッシュボードが起動しない | `bun install` を実行して chokidar と ws をインストール |
| fswatch が見つからない | macOS: `brew install fswatch`、Linux: `apt install inotify-tools` |
| QA レポートに 50 以上の問題がある | まず CRITICAL/HIGH に集中、残りは後で記録 |
